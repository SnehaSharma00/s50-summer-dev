create a basic express application with typescript.
make sure to follow these rules:
1. we will keep seperate modules.
    the path would like this src/app/<module-name>

2. inside every module we will follow this structure :
     <module-name>.routes.ts
     dto/<method>-<module-name>.routes.ts
     <module-name>controller.ts
     <module-name>.schema.ts

3.inside the src folder, we will have these folders:
    app
    constants
    middlewares
    utils
    config
    types
    logger

and these files:
    app.ts
    server.ts

4.use functional programming way for writing code
    you can use classes for specific usecasses like creating error class or logger class or..

5. use winston loger or loggin . use this logger utility everywhere to log entry / exit to controller, service and dal . also log exceptions and error. never use console.log() directly.

6. create response classes called ApiSuccessResponse and ApiErrorResponse.keep it in utils

7. create error classes and these errors will be thrown at required places . keep it in utils.
8 the error handling will be done in error handling middleware.

9. create a custom utility of asyncHandler to ease the error handling flow.

10. use zod for DTOs . infer typerscript types from zod schemas 

11. use drizzle orm with drizzle- kit for database migrations.

12. env will be extracted in config/env.config.ts

13. store the common typescript types in src/types

14. create zod validation middleware to be used for validating zod schemas and error handling middleware in src/middlewares.

15. Don't keep seperate env per environment.
    we will keep same .env for staging , testing, development, production. 
    Also, dont add any environment specific customisations anywhere in the code. 

16. Purpose of these files:
    app.js:
        -Register middlewares
        -Register routers

    server.js:
        -load env
        -database connections
        -create express app
        -start server

17. setup database connection to connect with supabase(postgres).

18. API response convention:
    Success:
        {
            success: true,
            message: "...",
            data: {...}
        }

    Error: 
        {
            success: false,
            message: "...",
            errors?: [...]
        }

19. Setup these middlewares:
    cors
    helmet
    express.json (body parser)
    cookie-parser
    morgan 

20. Have a health check API "/health"

21. Put the products router at "/api/v1/products"

22. Create simple CRUD REST APIs on products.

23. This project does not have any testing setup requirement.
